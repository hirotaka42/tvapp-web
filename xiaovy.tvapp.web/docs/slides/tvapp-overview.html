TVapp
複数の視聴サービスを、ひとつの体験に。

マルチサービス視聴・映画情報ポータル / Next.js 16 + Cloudflare + Firebase

STORY

背景から実装・運用までを、ひとつの物語で見る

1. 背景と中核

視聴サービスごとの分断と、サービスドックで切替える4ワールド。

2. 各ワールド

TVER、ABEMA、映画。YouTubeは準備中として同じ導線に置く。

3. 基盤と品質

映画D1基盤、アーキテクチャ、サービス群、テスト、運用。

このデッキはコード調査レポート inv1〜inv4 の事実に基づく自己完結HTMLです。

BACKGROUND

見たいもの、知りたいものがサービスごとに分断されている

TVER、ABEMA、映画情報はそれぞれ別の導線、別の画面、別の検索体験に分かれています。

視聴

無料見逃し、ライブ、VODで入口が分かれ、番組探しの文脈が切れる。

映画情報

上映中、近日公開、ニュース、ランキングを確認する導線が視聴体験と離れている。

横断

サービスをまたいで移動するたびに、ユーザーは今いる場所と目的を持ち直す。

TVappの狙いは、複数サービスを「ひとつのアプリ内の世界」として切替えられるようにすることです。

CORE MESSAGE

サービスドックで切替える、4つのワールド

TVER / ABEMA / 映画は稼働、YouTubeは準備中。data-svc とサービス状態で世界観ごと切替えます。

WORLD 1

TVERは、見逃し視聴の入口をホームに集約する

ランキング由来ヒーローから番組棚、終了間近までをたどり、エピソード再生へ進みます。

発見

ティッカー、カテゴリ、ランキング棚、終了間近帯で番組に出会う。

継続

レジュームと番組詳細導線で、途中からの視聴に戻る。

再生解決

純TypeScript resolverまたはAzure委譲の二段構え。非DRMのm3u8と字幕を返す。

Streaks playbackは Origin: https://tver.jp と月替わり key01..key06 を使う実装です。

WORLD 2

ABEMAは、ライブとVODを同じワールドで扱う

ライブ/VODヒーロー、いま放送中、VODランキング、EPG、番組棚から再生へ進みます。

ホーム

channels と slots を並列取得し、VODランキングも同じ画面に載せる。

BFF/API

channels、slots、VOD genres/ranking/program/series/episodeをBFF経由で取得。

再生

streaminglink、hls、key を持ち、license ticketを自前key routeへ書き換える。

slot再生は resolver 必須として扱う経路があり、外部API変更に備えた保全対象です。

WORLD 3

映画ワールドは「観る」前の情報体験に徹する

再生・予約・購入は扱わず、上映中/近日公開/ニュース/ランキングから映画を知る場所にします。

探す

上映中、近日公開、公開日未定、ジャンル別探索、現在の映画ランキング。

知る

映画ニュース、ヒーロー自動スライド、2:3ポスター、画像fallback。

残す

localStorageの「観たい」印と、予告編のYouTube検索導線。

映画は、視聴サービス横断アプリの中にある「情報ワールド」です。

CINEMA DATA

映画データ基盤は、鍵ゼロ・完全$0で日次更新する

映画.com、Filmarks、RSSをクロールし、認証付きingestでCloudflare D1へ投入します。

UI / UX

サービスごとに世界観を変えながら、操作の骨格は共通にする

切替の手触りは共通化し、背景・カード・ヒーロー表現でサービスごとの文脈を作ります。

TVER

ネオブルータリズム寄りの明快な棚構成。ランキングと終了間近を前面に置く。

ABEMA

暗いTV体験を基調に、ライブ、EPG、VODを同じ視線移動で扱う。

映画

デイライト・マチネ系。ヒーローカルーセル、ジャンルチップ、ポスター棚を中心にする。

レスポンシブは1280px基準の .wrap / .hd-in と、1000/980/640/430/400px付近の折返しで調整されています。

ARCHITECTURE

Next.jsのフロント+BFFを中心に、外部APIとD1へ接続する

認証はFirebase、配信・映画情報はBFF/APIで境界を切り、Cloudflare Workers/OpenNext上で動かします。

SERVICE MAP

サービス群は、現行Webを中心に役割が整理されている

tvapp-webがフロント+BFFの主戦場。Platform-Stream-Loaderは再生解決、tvapp-bffはレガシー資産として残っています。

QUALITY

テストは、外部サービスに触れる前の純関数に寄せている

Vitest + jsdom、JST固定、coverage 80%ゲートで、変化しやすい変換・正規化を固めます。

対象

TVER resolver/HLS proxy、ABEMA auth/VOD/streaminglink/hlsRewrite/正規化。

Cinema

status、crawler parser/filter/normalizer、映画クローラ独立Vitest config。

規模

検索上 describe/it/test は272件。live testは TVER_LIVE=1 がない限り除外。

coverage対象は src/lib/** と src/utils/**。閾値は80%です。

OPERATIONS

運用は、無料枠と安全なデプロイ導線に寄せる

Cloudflare無料枠、GitHub Actions想定、D1を使い、映画データは日次更新の自動運用に置きます。

完全$0

映画データ基盤は鍵ゼロ、Cloudflare D1とクローラで無料枠運用を狙う。

deploy:safe

版ゲート、再ビルド強制、build_sha 照合で古い成果物デプロイを防ぐ設計。

日次cron

Cinema契約書は .github/workflows/movie-ingest.yml による日次更新を想定。

このワークスペース直下では .github/workflows の実体は未検出。運用資料では「想定」と「現物」を分けて扱う必要があります。

NEXT

次は、横断体験を広げながら映画基盤の精度を上げる

現状は3ワールド稼働。映画は情報基盤ごと$0で自動運用できる形に近づいています。

YouTubeワールド

ready:false の準備中ワールドを、検索・視聴導線へ進める。

映画検索

home/ranking/newsの次に、映画単位の検索と詳細導線を配線する。

画像とジャンル

ポスター/ジャンル網羅の向上、R2画像キャッシュを検討する。

TVappは「サービスを増やす」だけでなく、「横断して迷わない体験」を育てる余地があります。

SUMMARY

TVappは、視聴と映画情報を同じ切替体験に載せる

体験

サービスドックでTVER、ABEMA、映画、YouTube準備中を切替える。

実装

Next.js 16 + Cloudflare Workers/OpenNext + Firebase + D1を中心に構成。

品質

Vitest約270+ケース、80%ゲート、純関数中心で変換ロジックを保護。

提案の核は「複数サービスをひとつの体験として切替えること」。映画ワールドはその拡張を情報提供側から担います。
