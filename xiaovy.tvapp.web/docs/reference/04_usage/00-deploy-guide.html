TVapp デプロイ手順書 — Cloudflare Workers / OpenNext

対象 tvapp-web version 1.6.2 区分 reference / 04_usage ステータス 現行手順

この手順書は、TVapp Web を Cloudflare Workers + OpenNext にデプロイする現行手順をまとめる。現行の構成、環境変数、D1 binding、鮮度照合の手順に合わせて記述する。

現行は Cloudflare Workers 上で Next.js 16 App Router を動かす。TVER/ABEMA のストリーム解決は Azure Function へ委譲でき、映画ワールドは Cloudflare D1 MOVIE_DB を読む。手動本番デプロイは npm run deploy:safe を使い、デプロイ後に /api/health の build_sha を照合する。

3. ローカル確認

  1. 依存を入れる。 
    npm install
  2. 通常テストを実行する。 
    npm run test
  3. 開発サーバーで確認する。 
    npm run dev
  4. Cloudflare 向け成果物を生成する。 
    npm run cf:build
  5. 必要に応じて OpenNext preview を起動する。 
    npm run cf:preview

4. 本番デプロイ

本番へ手動で出す場合は deploy:safe を使う。

npm run deploy:safe

scripts/deploy.sh は次の順に実行する。

  1. src、package.json、wrangler.jsonc、設定ファイル等に未コミット変更があれば中断する。
  2. .open-next と .next を削除する。
  3. npm run cf:build で再ビルドする。
  4. wrangler deploy --var BUILD_SHA:<HEAD> でデプロイする。
  5. 本番 /api/health を最大 12 回確認し、build_sha が HEAD と一致するか照合する。
opennextjs-cloudflare deploy や bare wrangler deploy は既存成果物を使う経路がある。手動本番では、古い成果物の混入を検出するため deploy:safe を使う。

6. 映画 ingest

映画データはアプリ本体のデプロイとは別に、日次クローラから /api/service/cinema/ingest へ投入する。

cd movie-crawler
node crawl.mjs --mode=post --endpoint=https://<host>/api/service/cinema/ingest --secret-env=INGEST_SECRET

ingest route は middleware では公開扱いだが、route 内で x-ingest-secret と INGEST_SECRET を定数時間比較する。D1 schema は db/cinema/schema.sql にある。
