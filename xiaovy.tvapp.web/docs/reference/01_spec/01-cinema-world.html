TVapp 映画ワールド仕様書

対象 tvapp-web / cinema 区分 reference / 01_spec データ Cloudflare D1 MOVIE_DB ステータス 実装済み

映画ワールドは、TVapp のサービスドックに追加された第4ワールドである。目的は映画情報の閲覧であり、アプリ内での映画再生、座席予約、チケット購入、配信購入の導線は持たない。

本書は現行実装に合わせて記述する。db/cinema/CONTRACT.md は一次資料だが、公開スケジュール API など現行コードに存在しない要素が含まれるため、本書では src/app/api/service/cinema、src/types/cinema.ts、db/cinema/schema.sql、movie-crawler の実装を優先する。

1. 目的と非対象

区分内容
目的上映中、近日公開、公開日未定、映画ニュース、ランキング、予告編検索を同一ワールドで閲覧できるようにする
対象ユーザー操作ジャンル絞り込み、ランキング切替、ニュース外部リンク、予告編 YouTube 検索、localStorage の「観たい」印
非対象映画本編の再生、映画館予約、チケット購入、配信購入、外部サービスのアカウント連携
サービス状態serviceCatalog.ts で cinema は ready: true

3. データフロー

4. D1 スキーマ

D1 binding は MOVIE_DB、database name は tvapp-movie。スキーマは db/cinema/schema.sql にある。

テーブル用途
movies作品本体。公開日、精度、上映中フラグ、ジャンル、ポスター、キービジュアル等
movie_credits監督、キャスト
movie_source_ids外部ソースの ID と URL
movie_news映画ニュース
news_movie_mapニュースと作品の対応
popularity_snapshotsClip、評価平均、評価件数などの時系列 snapshot
ingest_runs日次 ingest の実行記録

公開ステータスは D1 の列として固定保存せず、read 時に src/utils/cinema/status.ts で導出する。

status導出条件
undatedreleaseDate がない、または datePrecision === 'unknown'
postponedisPostponed が truthy
upcomingJST 今日より releaseDate が未来
now_showingnowShowing が truthy
ended上記に該当せず、公開日が過去または当日

5. クローラと正規化

項目現状
パッケージmovie-crawler。Node ESM、依存は cheerio と fast-xml-parser
取得源映画.com iCal/上映中/近日公開、MovieWalker list/detail、Filmarks now/coming、ナタリー/映画.com/Real Sound RSS
取得制御allow-list された host/path のみ。fetcher 側で同一 host 1から2秒間隔
除外興収、観客動員、ランキング、売上などを含むニュースは NEWS_BLOCK_PATTERN で除外
分類公開日、キャスト、予告、ポスター、映画祭、続編、リバイバル、舞台挨拶などのカテゴリを pattern で推定
映画文脈映画、劇場、公開、監督、主演などの語を文脈判定に利用
サムネイルニュース本文の og:image 等から補完。作品画像はポスター/キービジュアル URL を利用
出力--mode=post で ingest、--mode=sql で SQL 出力
 
cd movie-crawler
node crawl.mjs --mode=post --endpoint=https://<host>/api/service/cinema/ingest --secret-env=INGEST_SECRET
node crawl.mjs --mode=sql --out=seed.sql

6. API 仕様

API内容認証/制御
GET /api/service/cinema/home映画ホーム一括取得middleware の Bearer 存在チェック
GET /api/service/cinema/rankingランキング取得。type に応じて上映中/期待作middleware の Bearer 存在チェック
GET /api/service/cinema/newsニュース取得。category filter に対応middleware の Bearer 存在チェック
POST /api/service/cinema/ingestクローラからの投入middleware では公開、route で x-ingest-secret を照合

7. UI/UX

要素現行仕様
テーマhtml[data-svc="cinema"] と .cinema-world の CSS 変数で映画用の明るいデイライト/マチネ配色を使う。ダークテーマにも対応
ヒーロー最大6作品を自動スライド。ランキング上位、期待作、人気作品から重複排除して選ぶ
ジャンルレンズ2本以上の作品があるジャンルを選択肢にし、棚とランキングを絞り込む
カード2:3 ポスター、タイトル、公開日、ジャンル、監督/キャスト、人気指標を表示。画像失敗時はタイトル入り fallback
観たい印localStorage に slug 配列を保存。サーバ同期はしない
ニュース外部 URL が http/https の場合だけリンク化。関連作品がある場合は slug を持つ
予約/購入表示しない
