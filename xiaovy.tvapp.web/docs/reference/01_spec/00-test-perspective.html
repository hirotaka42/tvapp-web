TVapp テスト観点仕様書

対象 tvapp-web (xiaovy.tvapp.web) 関連 TVER / ABEMA / 映画 / BFF / 純関数 ランナー Vitest + jsdom ステータス 導入済み

本書は TVapp Web の現行テスト構成と、保守時に確認する観点を整理する。現行コードでは Vitest、coverage、live test の各設定が導入済みである。

現行の root テストは npm run test、coverage は npm run test:coverage、実 TVER 疎通を含むライブテストは npm run test:live で実行する。coverage gate は src/lib/** と src/utils/** に 80% 閾値を設定している。

1. テスト基盤

項目現状
ランナーvitest run
watchvitest
coveragevitest run --coverage、provider は v8
環境jsdom。日付ぶれ防止のため TZ=Asia/Tokyo
対象src/**/*.test.ts、src/**/*.test.tsx
live testTVER_LIVE=1 のときだけ *.live.test.ts を含める
coverage 対象src/lib/**、src/utils/**
coverage 閾値lines/functions/branches/statements すべて 80%
 
npm run test
npm run test:coverage
npm run test:live

3. TVER 観点

分類確認内容
sessionplatform uid/token の抽出、必須欠落時の失敗、上流ステータス伝播
episode/staticscontent.version 抽出、静的 episode JSON の streaks 情報取得
月替 keyJST 月から (month % 6) || 6 で key01..key06 を選ぶ。6月/12月は key06
playbackOrigin: https://tver.jp、X-Streaks-Api-Key、非 DRM m3u8 の採用
proxystreaks.jp 系 URL の allow-list、manifest URL の自 proxy 書き換え

4. ABEMA 観点

分類確認内容
homeView現在放送中、次番組、EPG grid、ticker、棚の導出。空配列、日跨ぎ、窓外 slot を含む
VODranking/program/series/episode の正規化と upstream failure の扱い
streaminglinkAzure 委譲、ライブ fallback、slot の resolver unavailable、reason 分類
HLS rewriteABEMA/Akamai 系 allow-list、m3u8 判定、abematv-license://ticket の key endpoint 置換
keysession id と ticket による鍵取得、鍵なし 404
